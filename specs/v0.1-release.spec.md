## 意图

记录 Vox v0.1 的实际实现状态，作为后续迭代的基线。此 spec 反映代码实际做了什么（而非原计划），所有场景均已验证通过。

## 已定决策

以下决策与原 project.spec.md 有偏差，以代码实际为准：

- 项目名: Vox (原 makepad-voice-input)
- 配置路径: `~/.config/vox/config.json` (原 `~/.config/makepad-voice-input/`)
- API 默认端口: 18080 (原 8080)
- 隐藏 Dock: 不调用 `show_in_dock(false)`（会导致 NSStatusItem 消失），.app bundle 用 `LSUIElement=true`
- HTTP 请求格式: JSON + base64 音频 (原 spec 为 multipart/form-data，实际 ominix-api 只接受 JSON)
- ASR language 参数: 发送全名 "Chinese"/"Japanese" (原 spec 为 ISO code "zh"/"ja")
- 波形动画: 脉冲呼吸圆点 (原 spec 为 5 根竖条 waveform，因 Splash 不能添加 instance 变量而改)
- ObjC FFI: 使用 `makepad_objc_sys`（与 Makepad 共享 NSApplication 实例），非 `objc2` crate
- 菜单栏图标: 文字 "MIC" (原 spec 为 PNG 图标，因 NSImage 模板图标在 Makepad 环境下不可靠)
- 入场/退场动画: 未实现 (Splash 不支持 instance 变量，Animator 方案受限)
- LLM system prompt: 包含翻译能力 + 目标语言标记 `[目标语言:xxx]` (原 spec 仅纠错)
- 文言文模式: 新增语言选项，原 spec 无
- 环境变量: 支持 `MOONSHOT_API_KEY` 等覆盖配置，原 spec 无

## 边界

### 已知限制（v0.1）
- 启动时 capsule 和 settings 窗口可能短暂可见（窗口隐藏机制不完善）
- 波形动画为脉冲圆点，非 5 根竖条（Splash 不支持添加 instance 变量）
- 无入场/退场动画（缺少 Animator 支持的 instance 变量）
- 菜单栏图标为文字 "MIC"，非图形图标
- 部分 emoji 不显示（✨ ⏳ 等）
- `show_in_dock(false)` 未使用（会隐藏 NSStatusItem）
- 热键配置 UI 未实现（代码中硬编码为左 Option）
- 文字宽度过渡动画未实现

### 排除范围
- Windows / Linux / Web 平台支持
- 流式转录（边说边出字）
- 自定义热键配置 UI
- 开机自启动
- 自动更新
